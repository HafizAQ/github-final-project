# Contributing to Simple Interest Calculator

Thank you for your interest in contributing to this project.

All contributions, bug reports, bug fixes, documentation improvements, enhancements, and ideas are welcome.

---

## Ways to Contribute

You can contribute in many ways, including:

- Reporting bugs
- Fixing issues
- Improving documentation
- Suggesting enhancements
- Sharing new ideas
- Refactoring code
- Improving user experience

---

## Contribution Guidelines

### Code Style

Follow the existing coding style and project structure to maintain consistency across the codebase.

### Documentation

Keep documentation clear and updated. Add comments where necessary and provide meaningful descriptions in pull requests.

### Testing

Test your changes before submitting a pull request. Ensure that all existing functionality works correctly.

### Bug Reports

When reporting bugs, include:

- A clear title and description
- Steps to reproduce the issue
- Expected behavior
- Actual behavior
- Screenshots or logs if applicable

### Enhancements and Ideas

Suggestions for new features or improvements are always encouraged. Open an issue to discuss major changes before starting work.

---

## Pull Request Process

1. Fork the repository
2. Clone your fork locally

```bash
https://github.com/HafizAQ/Simple_Interest_calculator_Q
```

3. Create a new branch
4. Make your changes
5. Commit your updates
6. Push the branch to GitHub
7. Submit a pull request

---

## Code Review

All contributions will go through a review process to help maintain code quality and project consistency.

Thank you for contributing to the Simple Interest Calculator project.
